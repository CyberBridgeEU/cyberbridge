🎯 WHAT WAS IMPLEMENTED
------------------------

✅ Global Authentication Middleware
   - Runs on ALL routes automatically
   - No need to add Depends() to every endpoint
   - Centralized auth logic in one place

✅ Principal Type Tracking
   - JWT Bearer token → principal.type = "user" (human users)
   - API Key → principal.type = "service" (automation)
   - Know if request is from web UI or automated script

✅ Automatic Logging
   - "👤 User john@example.com started scan" (JWT)
   - "🤖 Service automation@example.com started scan" (API Key)


📋 HOW IT WORKS
----------------

EVERY request goes through this flow:

   Request
      ↓
   [CORS Middleware]
      ↓
   [Auth Middleware] ← YOU ARE HERE
      ↓
   Is route public? (login, health, docs)
      ↓ No
   Has JWT token?
      ↓ Yes → Set principal.type = "user"
      ↓ No
   Has API Key?
      ↓ Yes → Set principal.type = "service"
      ↓ No
   REJECT 401 ❌
      ↓
   Your Endpoint
      ↓
   Access request.state.principal


🔑 PRINCIPAL OBJECT
-------------------

After successful auth, every request has:

request.state.principal = {
    "type": "user",              # or "service"
    "user": <User object>,
    "user_id": "uuid-here",
    "email": "user@example.com",
    "role": "admin",             # or "user"
    "auth_method": "jwt"         # or "api_key"
}


💻 USING IN ENDPOINTS
---------------------

Method 1 - Get full principal:
    from app.api.deps import get_principal
    
    @app.post("/scan")
    async def scan(request: Request):
        principal = get_principal(request)
        
        if principal["type"] == "user":
            print("Human user via web UI")
        else:  # "service"
            print("Automated service via API")

Method 2 - Get just the user:
    from app.api.deps import get_current_user_from_principal
    
    @app.post("/scan")
    async def scan(user: User = Depends(get_current_user_from_principal)):
        print(f"Request by {user.email}")

🔐 AUTHENTICATION METHODS
--------------------------

1. JWT Bearer Token (for users)
   Header: Authorization: Bearer <token>
   Type: principal.type = "user"
   Use: Web UI, manual operations
   
   curl http://localhost:8001/scans \
     -H "Authorization: Bearer eyJhbGc..."

2. API Key (for services)
   Header: X-API-Key: <key>
   Type: principal.type = "service"
   Use: Automation, scripts, CI/CD
   
   curl http://localhost:8001/scans \
     -H "X-API-Key: dk_prod_AbCdEf..."


🌐 PUBLIC ROUTES (No Auth Required)
------------------------------------

These routes DON'T require authentication:
  - /auth/login      (get JWT token)
  - /auth/register   (create user)
  - /auth/refresh    (refresh token)
  - /auth/info       (get auth config)
  - /health          (health check)
  - /docs            (API documentation)
  - /redoc           (ReDoc docs)

ALL OTHER ROUTES require authentication automatically!


🎓 WHY THIS IS BETTER
---------------------

Before (Per-endpoint dependency):
  ❌ Had to add Depends(get_current_user) to every endpoint
  ❌ Easy to forget and leave endpoints unprotected
  ❌ Couldn't tell if request was from user or service
  ❌ Auth logic scattered across endpoints

After (Global middleware):
  ✅ All routes protected by default
  ✅ One place for all auth logic
  ✅ Know if user or service made request
  ✅ Enterprise pattern (AWS, Google Cloud, etc.)


📊 LOGGING EXAMPLES
-------------------

The middleware logs every authentication:

Successful JWT (User):
  ✅ JWT Auth: admin@example.com (type=user) → POST /scan
  👤 User scan initiated by admin@example.com via jwt

Successful API Key (Service):
  ✅ API Key Auth: automation@example.com (type=service) → POST /scan
  🤖 Automated scan initiated by service (user: automation@example.com) via api_key

Failed Authentication:
  ❌ Unauthorized: POST /scan (no valid JWT or API key)
