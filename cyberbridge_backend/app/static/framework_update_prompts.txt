================================================================================
        CYBERBRIDGE - FRAMEWORK UPDATE PROMPT EXAMPLES
================================================================================

This guide shows you what to ask Claude when you need to update a framework
(CRA, ISO27001, or NIS2) with new questions, chapters, or objectives.

================================================================================
                            WHAT YOU CAN ADD/UPDATE
================================================================================

✅ CAN ADD:
-----------
1. NEW QUESTIONS
   - Question title
   - Question text
   - Which chapter it belongs to
   - Assessment type (conformity/audit)
   - Display order

2. NEW CHAPTERS
   - Chapter name
   - Chapter description
   - Chapter number
   - Display order

3. NEW OBJECTIVES
   - Objective title
   - Subchapter identifier (e.g., "3.2.1")
   - Which chapter it belongs to
   - Requirement description (what the requirement mandates)
   - Objective utilities (why this objective matters)

✅ CAN UPDATE (Existing Items):
--------------------------------
1. OBJECTIVE FIELDS
   - Objective title
   - Requirement description
   - Objective utilities
   - Compliance status

❌ CANNOT ADD/UPDATE:
---------------------
These are user-generated or organization-specific:
- Policies (created by users for their organization)
- Answers (user responses to assessment questions)
- Evidence (files uploaded by users)
- Assessments (created by users)
- Policy-Objective links (set by users)

================================================================================
                        IMPORTANT GUIDELINES
================================================================================

1. FRAMEWORK IDENTIFICATION
   ---------------------------
   Always specify which framework:
   - "CRA framework"
   - "ISO27001 framework"
   - "NIS2 framework"

2. CHAPTER REFERENCES
   --------------------
   Use clear chapter identification:
   - "Chapter 5" (for new questions)
   - "Subchapter 3.2.1" (for objectives)
   - If adding to new chapter, specify chapter number

3. REQUIRED INFORMATION
   ---------------------
   For Questions:
   - Question text (required)
   - Chapter ID or number (required)
   - Assessment type (conformity/audit) - defaults to conformity if not specified

   For Objectives:
   - Title (required)
   - Subchapter (required)
   - Requirement description (required)
   - Objective utilities (required)
   - Chapter ID (required)

   For Chapter:
   - Name (required)
   - Description (optional but recommended)
   - Chapter number (required)

4. UPDATE SCOPE
   -------------
   For updating existing objectives:
   - Identify by subchapter number
   - Specify exactly which fields to update
   - Other fields remain unchanged

5. ASSESSMENT TYPES
   -----------------
   Available types:
   - conformity (default) - Initial compliance check
   - audit - Formal audit assessment

================================================================================
                        TIPS FOR EFFECTIVE PROMPTS
================================================================================

✓ BE SPECIFIC: "Add to Chapter 5" not "Add some questions"
✓ BE COMPLETE: Include all required fields (title, description, utilities)
✓ BE CLEAR: Use quotes for exact text you want
✓ USE STRUCTURE: Number items if adding multiple elements
✓ REFERENCE EXISTING: Use subchapter numbers to update existing objectives

✗ DON'T BE VAGUE: "Add security questions" (which chapter? what questions?)
✗ DON'T ASSUME: If you don't specify assessment type, it defaults to conformity
✗ DON'ON'T FORGET UTILITIES: Every objective needs utilities (the "why")

================================================================================
                        UNDERSTANDING THE PROCESS
================================================================================

STEP 1: You provide the prompt (examples above)
        ↓
STEP 2: Claude creates a new update file (e.g., cra_update_2.py)
        ↓
STEP 3: File is automatically discovered by the system
        ↓
STEP 4: Users see "Update 2 - Available" in Framework Management
        ↓
STEP 5: Users click "Preview" to see changes
        ↓
STEP 6: Users click "Apply" to implement changes
        ↓
STEP 7: Update status changes to "Applied"

================================================================================
                        WHAT HAPPENS TO EXISTING DATA
================================================================================

When you apply an update:

✓ PRESERVED:
  - All existing questions remain
  - All existing objectives remain
  - All user answers remain
  - All policies remain
  - All policy-objective links remain
  - All assessment data remains

+ ADDED:
  - New questions appear in assessments
  - New objectives available for policy linking
  - New chapters organize content better

↻ UPDATED:
  - Modified objectives show new descriptions
  - Users see updated requirement text
  - Objectives maintain same ID (links preserved)

✗ NEVER DELETED:
  - Updates never remove existing data
  - Only additions and modifications
  - All historical data preserved

================================================================================
                        TROUBLESHOOTING
================================================================================

PROBLEM: "Update doesn't appear in UI"
SOLUTION: Make sure you selected exactly ONE framework in the dropdown

PROBLEM: "Preview shows nothing"
SOLUTION: Check that the update file was created correctly and framework name matches

PROBLEM: "Apply fails"
SOLUTION: Check chapter IDs exist, verify all required fields are present

PROBLEM: "Need to add to a chapter that doesn't exist yet"
SOLUTION: First add the chapter, then add questions/objectives in the same update

================================================================================
                        VERSION TRACKING
================================================================================

Each update gets an incremental version number:
- First update: version 1 (e.g., cra_update_1.py)
- Second update: version 2 (e.g., cra_update_2.py)
- Third update: version 3 (e.g., cra_update_3.py)

Version numbers are per-framework:
- CRA: cra_update_1.py, cra_update_2.py, ...
- ISO27001: iso27001_update_1.py, iso27001_update_2.py, ...
- NIS2: nis2_update_1.py, nis2_update_2.py, ...
